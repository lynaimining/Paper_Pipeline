## 六、问题诊断与改进方案

### 问题1：矿区级精度偏低（12%）

**根本原因**：
1. Qwen视觉识别的图片文字粗略（如"115°E"而非"115°12'45\"E"）
2. LLM从caption推断坐标，无法读取精确网格
3. 大多数论文图件只标注到度数

**改进方案**：

#### 方案A：Qwen专项优化（推荐）
```python
# 让Qwen专门提取坐标网格
qwen_prompt = """
识别地质图中的坐标信息：
1. 坐标轴刻度（精确到度分秒）
2. 坐标网格线间隔
3. 图中标注的经纬度数字
4. 矿体/矿区位置相对坐标网格的位置

输出格式：
- 左下角坐标: (115°10'30"E, 35°25'15"N)
- 右上角坐标: (115°15'20"E, 35°30'10"N)
- 矿区中心估计: (115°12'45"E, 35°27'30"N)
"""
```

**预期效果**：矿区级精度提升至30-50%

#### 方案B：置信度校准（立即可做）
```python
# 基于实测数据调整置信度
if extraction_method == "图件坐标网格" and confidence >= 0.7:
    # 降级置信度（因为LLM过于乐观）
    adjusted_confidence = confidence * 0.8
    precision_note = "省级（图件网格粗糙）"

if extraction_method == "明确经纬度":
    # 保持高置信度
    precision_note = "矿区级"
```

#### 方案C：外部地理编码（补充）
```python
# 对无精确坐标但有详细地名的论文
# 调用地理编码API（如Google Geocoding）
geocode_api("Mount Evelyn deposit, Pine Creek, Northern Territory, Australia")
# 返回：(-13.58333, 131.83333, precision="city_level")
```

### 问题2：置信度与精度弱相关

**根本原因**：LLM对"图件坐标网格"的理解与实际不符

**改进方案**：
```python
# 明确要求LLM区分坐标精度
prompt += """
置信度评分必须与精度一致：
- 矿区级精度（误差<5km）→ 置信度≥0.9
- 省级精度（误差10-100km）→ 置信度0.6-0.8
- 国家级精度（误差>100km）→ 置信度<0.6

如果图件坐标网格只到度数（无分秒），
即使网格清晰，也只能给"省级"精度 + 0.7置信度
"""
```
